# UWI Principal's Research Awards & Festival - Application Workflows

## Overview

This document outlines the complete workflows and user journeys for all stakeholder roles in the UWI Principal's Research Awards & Festival application. Each workflow describes the key navigation paths, available actions, and information access points specific to that user role.

---

## 1. Admin Workflow

**Purpose**: Oversee entire festival operations, manage workflow approvals, coordinate scheduling, and generate reports.

### User Journey

```
Login → Admin Dashboard → View Submissions → Monitor Reviews → Approve Abstracts → Manage Agenda → View Results → Generate Digest
```

### Key Actions

1. **Login & Access Control**
   - Access comprehensive admin dashboard
   - View system-wide metrics and status
   - Monitor all submission and review stages

2. **View All Submissions**
   - Access complete submission database
   - Filter by status, theme, author, submission date
   - Search for specific submissions
   - Bulk actions on multiple submissions
   - Track submission progress through workflow

3. **Monitor Review Progress**
   - Track review completion rates
   - See which abstracts are pending review
   - Monitor reviewer assignment and workload
   - Access consolidated reviews for each abstract
   - Generate review statistics and reports

4. **Approve/Finalize Abstracts**
   - Make final editorial decisions:
     - Accept for Oral Presentation
     - Accept for Poster Session
     - Request Major Revisions
     - Request Minor Revisions
     - Reject submission
   - Send decision notifications to authors
   - Archive rejected submissions appropriately
   - Confirm final list of approved presentations

5. **Manage Festival Agenda**
   - Create presentation tracks and sessions
   - Schedule approved oral presentations:
     - Assign time slots
     - Assign venues/rooms
     - Assign session chairs
     - Manage conflicts and constraints
   - Organize poster sessions:
     - Assign poster board locations (e.g., A12, B5)
     - Group by theme or track
   - Generate printable schedules
   - Publish updated schedules to attendees

6. **Monitor Judging Process**
   - Track judge submissions and completion
   - Monitor scoring progress
   - View interim and final rankings
   - Resolve scoring discrepancies if needed

7. **View Results & Awards**
   - See final judging scores for oral presentations
   - See final judging scores for poster presentations
   - Generate award winner lists
   - Prepare award certificates
   - Plan award ceremony details

8. **Generate Digest & Reports**
   - Create conference digest/summary document:
     - List of accepted presentations
     - Presenter information
     - Session schedule
     - Award winners
     - Key statistics and highlights
   - Generate various reports:
     - Submission statistics
     - Review metrics
     - Attendance analytics
     - Feedback summaries
   - Export data for post-conference analysis

### Key Information Access

- Complete submission database
- All reviews and reviewer feedback
- Review approval status
- Judging scores and results
- Approved presentation list
- Schedule and venue information
- Attendee registration data
- System-wide statistics and analytics
- Report generation tools

## 2. Author Workflow

**Purpose**: Submit research for consideration, track submission progress, and prepare presentation materials.

### User Journey

```
Sign Up → Create Submission → Track Status → Receive Feedback → Prepare Presentation → View Assigned Details
```

### Key Actions

1. **Account Creation & Registration**
   - Create account with email and password
   - Complete author profile information:
     - Name and affiliation
     - Discipline/research area
     - Contact information
   - Set research interests and track preferences

2. **Create & Submit Abstract**
   - Fill in submission form:
     - Research title
     - Authors and affiliations
     - Abstract text (within word limit)
     - Keywords and research themes
     - Select primary and secondary tracks
   - Upload supplementary materials (optional):
     - Full paper or extended abstract
     - Supporting figures/data
     - References
   - Review and submit

3. **Track Submission Status**
   - Dashboard showing all submissions
   - Real-time status updates:
     - **Under Review** - assigned to reviewers
     - **Review Complete** - awaiting editorial decision
     - **Changes Requested** - revise and resubmit
     - **Accepted** - approved for presentation
     - **Rejected** - not accepted (with feedback)
     - **Recommended for Poster** - suggested format change

4. **View Reviewer Feedback**
   - Access consolidated reviewer comments
   - Review suggestions for improvement
   - Understand reasons for decisions
   - Identify requested changes

5. **Resubmit if Requested**
   - Access submission form with prior feedback
   - Revise abstract and materials
   - Resubmit with revision notes
   - Provide rebuttal to reviewer comments (optional)

6. **View Accepted Presentation Details**
   - Confirmation of acceptance
   - Assigned presentation format:
     - **Oral Presentation**:
       - Scheduled date, time, and venue
       - Track and session information
       - Session chair name
       - Presentation duration (minutes)
     - **Poster Presentation**:
       - Assigned poster board location (e.g., A12)
       - Session date and time
       - Poster board dimensions
       - Setup and takedown schedule
   - Download presentation guidelines
   - Access presenter resources:
     - Slide templates
     - Poster templates
     - Technical requirements
     - Copyright and permissions info

7. **Prepare Presentation Materials**
   - Create slides (for oral presentations)
   - Design poster (for poster presentations)
   - Follow conference formatting guidelines
   - Submit final materials (if required)
   - Test technical setup (for oral presentations)

8. **Access Pre-Conference Information**
   - Download final program with own presentation
   - Access venue maps and directions
   - View speaker logistics information
   - Confirm final attendance

### Key Information Access

- Personal submission dashboard
- Submission guidelines and templates
- Reviewer feedback and comments
- Presentation assignment details
- Presenter guidelines and resources
- Conference schedule
- Final program materials

## 3. Reviewer Workflow

**Purpose**: Review submitted abstracts, provide feedback, and approve or request changes to research submissions.

### User Journey

```
Login → Dashboard → Assigned Reviews → View Abstract Details → Provide Review → View Results
```

### Key Actions

1. **Login & Authentication**
   - Access the application using reviewer credentials
   - Navigate to Reviewer Dashboard

2. **View Assigned Submissions**
   - Filter submissions by assigned theme
   - See submission status (pending review, under review, decision pending)
   - Access abstract documents and supplementary materials

3. **Review Abstracts**
   - Read author information and research details
   - View prior reviews (if applicable)
   - Provide comprehensive feedback on:
     - Research quality and innovation
     - Methodological soundness
     - Relevance to conference themes
     - Presentation clarity

4. **Submit Review Decision**
   - Choose action: Approve, Request Changes, or Recommend for Alternative Format
   - Provide detailed comments for authors
   - Submit review recommendation

5. **View Review Progress**
   - Monitor abstract review status
   - See when all reviewers have submitted decisions
   - Access consolidated feedback before final editorial decision
   - Confirm acceptance of abstracts (once approved by editors)

6. **Map Reviewers** (Editor Only)
   - Assign reviewers to submissions
   - Balance reviewer workload
   - Track review completion rates

### Key Information Access

- Theme-specific submissions
- Abstract content and supplementary files
- Author contact information
- Review history and previous feedback
- Conference themes and submission guidelines
- Reviewer guidelines and evaluation rubrics

## 4. Public/Visitor Workflow

**Purpose**: Learn about the festival, view program, and optionally register as an attendee.

### User Journey

```
Landing Page → Browse Schedule → View Digest & Awards → Explore Research → Sign Up (Optional) → Access Attendee Features
```

### Key Actions

1. **Access Landing Page**
   - Overview of festival theme and purpose
   - Key dates and important information
   - Featured presentations and speakers
   - Quick links to navigation

2. **View Festival Schedule**
   - Browse complete agenda without registration
   - See presentations by:
     - Date and time
     - Track/theme
     - Presentation type
   - View venue information
   - Filter presentations by interests

3. **Explore Presentations**
   - View presentation titles and abstracts
   - See presenter information
   - Read research summaries
   - Discover related presentations

4. **View Conference Digest**
   - Access published conference summary
   - See accepted presentations list
   - View author information
   - Explore research highlights

5. **View Awards & Winners**
   - See award categories
   - View award winners
   - Read winner biographies
   - Explore winning research

6. **Sign Up for Attendee Account** (Optional)
   - Create account to access additional features
   - Confirm email and complete profile
   - Access attendee-only resources

7. **Access Attendee Features** (After Signup)
   - RSVP to presentations
   - Create personal schedule
   - Get QR code for check-in
   - Participate in Q&A
   - Provide session feedback

### Key Information Access

- Festival overview and key details
- Complete public schedule
- Presentation abstracts and information
- Presenter information and affiliations
- Venue maps and locations
- Conference digest
- Awards and recognition information
- Sign-up forms and registration

## 5. Attendee Workflow

**Purpose**: Participate in the festival, attend sessions, network, and engage with research presentations.

### User Journey

```
Login → Dashboard → View Agenda → Select Sessions → RSVP → Access Session Details → Attend → Give Feedback
```

### Key Actions

1. **Login & Registration**
   - Create account or login with credentials
   - Complete profile information
   - Confirm attendance status

2. **Browse Festival Agenda**
   - View complete conference schedule
   - See all presentations organized by:
     - Day and time slots
     - Track/theme
     - Presentation type (oral vs. poster)
     - Venue/location

3. **Explore Session Details**
   - View presentation titles and abstracts
   - See presenter information and affiliations
   - Check session location and venue details
   - Review presenter biographies
   - See related presentations in same track

4. **RSVP to Sessions**
   - Mark sessions as "Interested"
   - Confirm attendance for specific presentations
   - Build personal conference schedule
   - Receive reminders before sessions

5. **Generate QR Code**
   - Access personal QR code for event check-in
   - Display code at registration and session entry points
   - Track attendance automatically

6. **Access Session Content**
   - View live presentation materials
   - Participate in Q&A sessions
   - Access presentation slides (if available)
   - Connect with other attendees

7. **Provide Session Feedback**
   - Rate presentations on:
     - Content quality
     - Presentation effectiveness
     - Relevance to interests
   - Leave written feedback and comments
   - Suggest improvements or follow-up topics

### Key Information Access

- Complete festival schedule
- Presentation titles, abstracts, and speakers
- Venue maps and location details
- Track descriptions and theme information
- Presenter profiles and contact information
- Personal schedule and RSVP status
- Session recordings (if available post-event)
- Feedback forms and survey links

## 6. Judge Workflow

**Purpose**: Evaluate presentations (oral and poster) at the festival and determine award winners.

### User Journey

```
Login → Judge Dashboard → Assigned Presentations → View Presentation Details → Submit Judging → View Results
```

### Key Actions

1. **Login & Authentication**
   - Access application using judge credentials
   - Navigate to Judge Dashboard

2. **View Assigned Presentations**
   - See list of presentations assigned for judging:
     - **Oral Presentations** (by session, time, track)
     - **Poster Presentations** (by location/board number)
   - Filter by presentation type or schedule

3. **Review Presentation Information**
   - Author(s) and institutional affiliation
   - Abstract and research overview
   - Presentation title and session information
   - Track category and theme
   - Scheduled time and venue

4. **Attend & Evaluate Presentations**
   - Attend live oral presentations or visit poster sessions
   - Take notes during presentations
   - Ask clarifying questions to presenters

5. **Submit Judging Scores**
   - Provide scores on evaluation criteria:
     - Research quality and significance
     - Clarity of presentation
     - Innovation and originality
     - Response to questions
     - Overall impact
   - Add commentary and notes
   - Submit judging decision

6. **View Judging Results**
   - See aggregated scores across all judges
   - View ranking of presentations
   - Identify top-ranked oral presentations
   - Identify top-ranked poster presentations
   - Confirm final award winners

### Key Information Access

- Complete list of assigned presentations
- Presentation schedules and venues
- Presenter information
- Abstract and research details
- Judging criteria and scoring guidelines
- Previous judging feedback
- Results and rankings dashboard

## 7. Usher Workflow

**Purpose**: Facilitate session check-in and ensure smooth participant flow during presentations.

### User Journey

```
Login → Session Assignment → Check-In Dashboard → Search Attendees → Scan QR Codes → Track Attendance
```

### Key Actions

1. **Login & Access Session Assignment**
   - Access usher dashboard with assigned sessions
   - See session details:
     - Session title and schedule
     - Venue and room location
     - Expected attendance
     - Setup time

2. **Prepare for Session**
   - Arrive early to session venue
   - Review expected attendees
   - Set up check-in station
   - Test QR code scanner

3. **Check-In Attendees - Method 1: QR Code Scanning**
   - Attendee presents QR code
   - Scan code with mobile device or dedicated scanner
   - System automatically records attendance
   - Attendee receives welcome confirmation
   - Mark as checked-in in system

4. **Check-In Attendees - Method 2: Manual Search**
   - Attendee provides name
   - Search attendee database
   - Locate attendee record
   - Manually mark as present
   - Provide confirmation

5. **Track Session Attendance**
   - View real-time attendance count
   - Monitor check-ins throughout session
   - Generate attendance report at session end
   - Note any late arrivals
   - Document attendance issues

6. **Session Management Tasks**
   - Announce session start time
   - Direct attendees to seating
   - Assist with technical issues
   - Manage session capacity if needed
   - Provide session information to attendees

7. **Post-Session Activities**
   - Generate final attendance report
   - Submit attendance data
   - Return equipment if applicable
   - Notes on session flow/issues

### Key Information Access

- Assigned session list and schedule
- Attendee database and QR codes
- Check-in status and tracking
- Session details and logistics
- Attendance reports
- Contact information for session organizers

## 3. Status Workflows

### Submission Status Flow

```
Draft (unsaved)
    ↓
Submitted
    ↓
Under Review (assigned to reviewers)
    ↓
    ├→ Changes Requested → Author Revises → Resubmitted → Under Review
    ├→ Accepted (Oral) → Scheduling → Scheduled
    ├→ Accepted (Poster) → Scheduling → Scheduled
    └→ Rejected → Archived
```

### Presentation Status Flow

```
Approved for Presentation
    ↓
Scheduled (assigned date/time/venue)
    ↓
Submitted for Judging
    ↓
Scored by Judges
    ↓
Results Finalized
    ↓
├→ Award Winner
├→ Top Ranked
└→ Presented
```

## 5. Key Features by Workflow Stage

### Pre-Festival Phase
- **Authors**: Submit abstracts
- **Reviewers**: Evaluate and provide feedback
- **Editors/Admin**: Manage review process, make decisions
- **Authors**: Resubmit if requested
- **Admin**: Schedule approved presentations
- **Judges**: Assigned to presentations
- **Public**: View schedule and register as attendee

### Festival Phase
- **Presenters**: Deliver oral presentations or present posters
- **Judges**: Evaluate presentations and submit scores
- **Attendees**: RSVP, attend sessions, provide feedback
- **Ushers**: Check in attendees, manage sessions
- **Admin**: Monitor all activities, handle issues

### Post-Festival Phase
- **Judges**: View final results and rankings
- **Admin**: Finalize results, generate digest, create reports
- **Authors**: View presentation feedback and outcome
- **Attendees**: Access recordings and continued resources

## 7. Security & Access Control

### Authentication Requirements
- All users except Public visitors must authenticate
- Role-based access control (RBAC) enforces permissions
- Each role can only access appropriate modules and data

### Data Visibility
- **Admin**: Full visibility of all data
- **Reviewers**: Only see assigned abstracts
- **Authors**: Only see their own submissions
- **Judges**: Only see assigned presentations
- **Attendees**: Only see public program information
- **Ushers**: Only see assigned session details
- **Public**: Limited to published information
